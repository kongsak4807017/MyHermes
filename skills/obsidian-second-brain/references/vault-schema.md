# Vault Schema

## Directory Structure

```
vault/
├── _HERMES.md              # Operating manual for Hermes
├── index.md                # Catalog of all pages (Hermes reads FIRST)
├── log.md                  # Activity timeline
├── SOUL.md                 # Identity, values, communication style
├── CRITICAL_FACTS.md       # ~120 tokens always loaded
├── PINNED.md               # Session-specific pinned context
├── raw/                    # IMMUTABLE source material
│   ├── urls/
│   ├── pdfs/
│   ├── audio/
│   └── images/
├── wiki/                   # Hermes workspace
│   ├── entities/           # People, companies, tools
│   ├── concepts/           # Ideas, frameworks, synthesis
│   ├── projects/           # Project notes
│   ├── daily/              # Daily notes (YYYY-MM-DD.md)
│   ├── logs/               # Work session logs
│   ├── reviews/            # Weekly/monthly reviews
│   ├── tasks/              # Task notes
│   └── decisions/          # ADRs
├── boards/                 # Kanban boards
│   ├── main.md
│   └── projects/
├── templates/              # Note templates
│   ├── daily.md
│   ├── project.md
│   ├── person.md
│   ├── idea.md
│   └── task.md
└── Research/               # Research outputs
    ├── Web/
    ├── X/
    ├── YouTube/
    └── NotebookLM/
```

## Key Files

### _HERMES.md
Operating manual. Loaded first on every vault operation.

### index.md
Page catalog. Hermes reads this to know what exists without loading everything.

### log.md
Activity timeline. Last 10 entries loaded at L1.

### SOUL.md
Identity file. Who the user is, communication style, thinking preferences.

### CRITICAL_FACTS.md
Always-loaded context. ~120 tokens.
- Timezone
- Manager/team
- Location
- Company/role
- Key preferences

### PINNED.md
Session-specific context. Loaded at L0 alongside SOUL.md.
- Complex task facts
- Schemas
- Reference data
- Cleared when task done

## Note Organization

### Entities
People, companies, tools. Linked from everywhere.

### Concepts
Ideas, frameworks, synthesis pages. Auto-generated by `/hermes-synthesize`.

### Projects
Active work. Status tracked, linked to people, tasks, decisions.

### Daily
One note per day. Links to everything that happened.

### Logs
Work session records. Link to projects and people.

### Reviews
Weekly/monthly structured reviews.

### Tasks
Individual task notes. Linked to projects and boards.

### Decisions
ADRs. Standalone or inside project notes.
