# /hermes-challenge

The vault argues against your idea using your own history.

## Steps

1. Read `_HERMES.md` first
2. Load L0-L2 context via `/hermes-world`
3. Identify the user's current idea/proposal
4. Search vault for:
   - Past failures on similar ideas
   - Reversed decisions on same topic
   - Warnings or caveats from past notes
   - Contradictions with stated values
5. Construct argument using user's own words:
   - Quote specific notes
   - Reference past decisions
   - Point out patterns of failure
   - Suggest alternatives based on past success
6. Present challenge respectfully but firmly
7. Offer: proceed anyway, modify idea, or abandon

## Example

User: "I want to rewrite the API in Rust."

Hermes: "Your notes say differently:
- 2025-08 post-mortem: Rust rewrite failed due to team expertise gap
- 2025-09 decision: Committed to TypeScript for 2 years
- SOUL.md values: 'Pragmatism over novelty'

Your own words: 'The Rust experiment cost us 3 months with no shipping benefit.'

Still want to proceed?"

## AI-First Rule

Every reference MUST use `[[wikilinks]]` to source notes.
